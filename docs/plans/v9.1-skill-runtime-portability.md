# Plan: v9.1.0 — Generic Skills-Based Runtime Portability

## Context

Agent Brain v9.0 shipped multi-runtime support (Claude, OpenCode, Gemini) via a `RuntimeConverter` protocol pattern. However, runtimes like **Codex** don't support Claude-style commands or agents — they only support **skills**. This milestone adds:

1. A **generic `skill-runtime` converter** that transforms all plugin artifacts into skill directories
2. A **Codex named adapter** as a predefined specialization
3. A **`--dir` option** for targeting arbitrary skill directories (Qwen, Cursor, etc.)

This makes Agent Brain portable to any runtime with skill directory support.

## Key Design Decisions

- `--agent skill-runtime` (explicit, avoids confusion)
- `--dir` is **required** for skill-runtime (no sensible default)
- Codex adapter is a named preset built on skill-runtime converter
- Commands become individual skill directories with SKILL.md + shell wrappers
- Agents become orchestration skill directories referencing dependent skills
- Templates/scripts become asset directories

## Transformation Rules

```
command agent-brain-init.md → <dir>/agent-brain-init/SKILL.md + scripts/init.sh
agent research-assistant.md → <dir>/agent-brain-research/SKILL.md (orchestration)
skill using-agent-brain/    → <dir>/agent-brain-using/SKILL.md + references/
templates/*                 → <dir>/agent-brain-setup/assets/
scripts/*                   → <dir>/agent-brain-verify/scripts/
```
