## ⚡ Elevated Concern — CLI-MCP-04 DoD Anchor (verified at code level)

**Finding:** the stdio MCP transport does not propagate the state directory to the subprocess it spawns.

**Chain (confirmed by reading the code):**
```
agent-brain --transport mcp --mcp-transport stdio query "X"
  → transport.py:open_backend() → McpStdioBackend(command="agent-brain-mcp")   # no forward_env, no --state-dir
  → _stdio_params() → args = ["--transport","stdio"]  (no --state-dir)
                    → env  = _effective_env()  → filtered to DEFAULT_ENV_ALLOWLIST
                             {PATH,HOME,USER,LANG,LC_ALL,TERM}   # AGENT_BRAIN_STATE_DIR dropped
  → agent-brain-mcp/config.py:_resolve_state_dir()
       --state-dir arg? no → AGENT_BRAIN_STATE_DIR env? no → cwd fallback
  → MCP server queries the corpus at cwd/.agent-brain, NOT the caller's intended state dir
```

**Impact:**
- ✅ Works on the **documented happy path**: CLI run from the project root with the default state dir → cwd fallback lands on the same `.agent-brain` the UDS leg uses → byte-identical holds.
- ⚠️ **Silently wrong** when `AGENT_BRAIN_STATE_DIR` is set (multi-instance / `AGENT_BRAIN_MODE`) **and** cwd ≠ project root → the MCP leg resolves a *different* backend than `--transport uds`. No error; just different results — directly contradicting the CLI-MCP-04 guarantee in that configuration.

**Why it wasn't caught:** the DoD-anchor contract test (`test_transport_equivalence.py`) `pytest.skip`s without `OPENAI_API_KEY` + binaries, so the full byte-compare does not run in keyless `task before-push`. The test passes `AGENT_BRAIN_STATE_DIR` to the CLI env but does not pin cwd, so even with keys the MCP leg's cwd fallback may not align with the seeded tmp corpus.

**Fix (small, well-scoped):** in `transport.py:open_backend()` (and `open_mcp_backend()`), construct the stdio backend with either:
- `McpStdioBackend(command=["agent-brain-mcp", "--state-dir", str(resolved_state_dir)])`, or
- `McpStdioBackend(command="agent-brain-mcp", forward_env=["AGENT_BRAIN_STATE_DIR", "AGENT_BRAIN_UDS_PATH", "AGENT_BRAIN_URL"])`

…matching what the framework harnesses already do (they pass `--backend uds --state-dir <path>` explicitly). Then harden the contract test to pin cwd to a non-project dir so the compare actually exercises override-based discovery.

## Resolution (2026-06-14 — fast-follow fix applied)

The elevated CLI-MCP-04 concern was closed in the same milestone branch, test-first (TDD):

- **Fix** (`agent-brain-cli/agent_brain_cli/client/transport.py`): new `_stdio_command(state_dir)` helper threads the resolved state dir into the spawned `agent-brain-mcp` as an explicit `--state-dir <path>` argument, with `AGENT_BRAIN_STATE_DIR` taking precedence over project-root discovery (mirroring the UDS leg). Both `open_backend()` (tools surface) and `open_mcp_backend()` (prompts/resources surface) use it. Falls back to the bare command only when no state dir resolves. The arg survives `DEFAULT_ENV_ALLOWLIST` filtering — the root cause of the divergence.
- **Regression tests** (`tests/test_transport_selector_mcp.py`, `TestMcpStdioStateDirForwarding`, 4 tests): assert the stdio backend carries `--state-dir` on both factories, that the env override wins over discovery (the exact audit failure scenario, reproduced RED→GREEN), and that no bogus `--state-dir` is passed when unresolvable.
- **Contract-test hardening** (`tests/contract/test_transport_equivalence.py`): the byte-identical DoD test now runs both legs from an isolated `tempfile.TemporaryDirectory()` cwd with no discoverable `.agent-brain`, so the MCP leg can only reach the seeded corpus via the propagated state dir — a cwd-fallback would have masked the bug. (Still gated on `OPENAI_API_KEY`; the hardening takes effect when keys are present in CI.)

**Gate:** `task before-push` green (544 MCP + 557/562 CLI + server/uds, 0 failures). The remaining test-coverage caveat (byte-compare still skips keyless locally) is unchanged from the original design decision but is now correctly exercised under override when keys are present.
