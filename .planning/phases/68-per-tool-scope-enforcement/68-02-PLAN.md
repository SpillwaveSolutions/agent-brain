<objective>
Enforce per-tool OAuth scope on the HTTP `/mcp` path (OAUTH-06 SC#1/#2/#3) with a
**pre-dispatch ASGI guard**. The MCP low-level server's `_handle_request()` catches ALL
handler exceptions and converts them to a JSON-RPC error inside an HTTP **200** envelope
(confirmed: `mcp/server/lowlevel/server.py` ~lines 785-788 `except Exception as err: ...
response = types.ErrorData(...)`), so an exception raised at the dispatch layer can NEVER
become an HTTP 403. Therefore the 403 MUST be decided and emitted BEFORE the request reaches
the session manager. The guard buffers the JSON-RPC body, maps `method`(+`params.name`) →
required scope, reads the caller's granted scopes off `scope["user"].scopes` (set by Phase
67's `AuthenticationMiddleware`), and on insufficient scope emits HTTP **403** with
`WWW-Authenticate: Bearer error="insufficient_scope", scope="<REQUIRED>",
resource_metadata="<PRM url>"` — never 401 — without calling the downstream app.

A defense-in-depth `_enforce_scope` also lands at the server.py dispatch points (call_tool,
read_resource, handle_subscribe, get_prompt) to cover in-process / stdio callers the HTTP
guard does not sit in front of.

Purpose: Middleware "is-authenticated" (Phase 67) cannot scope per-tool — the tool name is
only known from the request body. This closes design Risk 4 (per-tool scope escalation).
Output: `ScopeEnforcementMiddleware` (pre-dispatch guard) wired INSIDE
`AuthenticationMiddleware` in http.py + defense-in-depth `_enforce_scope` in server.py +
acceptance tests (incl. a real-HTTP-403 TestClient integration) using read-only tokens.
</objective>

    2. Add `class ScopeEnforcementMiddleware`:
    ```python
    class ScopeEnforcementMiddleware:
        """Pre-dispatch per-tool OAuth scope guard for POST /mcp (design Risk 4).

        Buffers the JSON-RPC request body, maps method(+params.name) -> required scope,
        reads the caller's granted scopes off scope["user"].scopes (set by the wrapping
        AuthenticationMiddleware), and on insufficient scope emits HTTP 403 +
        WWW-Authenticate insufficient_scope WITHOUT calling the downstream app. On
        sufficient scope (or a non-scoped method) it replays the buffered body to the
        downstream app. Engages only in oauth mode.
        """

        def __init__(self, app, *, resource_metadata_url: str) -> None:
            self.app = app
            self.resource_metadata_url = resource_metadata_url

        async def __call__(self, scope, receive, send) -> None:
            # Pass through anything that isn't an in-oauth-mode POST to /mcp.
            if (
                scope["type"] != "http"
                or scope.get("method") != "POST"
                or resolve_auth_mode() != AuthMode.oauth
            ):
                await self.app(scope, receive, send)
                return

            # 1) Buffer the full request body (handle multi-part http.request frames).
            body = b""
            more = True
            messages: list[dict] = []
            while more:
                message = await receive()
                messages.append(message)
                if message["type"] == "http.request":
                    body += message.get("body", b"")
                    more = message.get("more_body", False)
                elif message["type"] == "http.disconnect":
                    more = False

            # 2) Build a replay receive() so the downstream app still gets the body.
            replay = iter(messages)

            async def replay_receive():
                try:
                    return next(replay)
                except StopIteration:
                    # After buffered frames are exhausted, defer to the real receive
                    return await receive()

            # 3) Decide the required scope from the JSON-RPC body.
            required = self._required_scope(body)
            if required is None:
                # non-scoped method / unparseable / unknown tool -> let dispatch handle it
                await self.app(scope, replay_receive, send)
                return

            # 4) Read granted scopes set by AuthenticationMiddleware.
            user = scope.get("user")
            granted = list(getattr(user, "scopes", []) or [])

            # 5) Enforce. require_scope raises InsufficientScopeError on failure.
            try:
                require_scope(required, granted)
            except InsufficientScopeError as exc:
                await self._send_403(send, required=exc.required)
                return

            # 6) Sufficient scope -> downstream with the replayed body.
            await self.app(scope, replay_receive, send)

        def _required_scope(self, body: bytes) -> str | None:
            try:
                payload = json.loads(body)
            except (ValueError, TypeError):
                return None
            # JSON-RPC batch is a list; scope-check is per the MCP single-request shape.
            if not isinstance(payload, dict):
                return None
            method = payload.get("method")
            params = payload.get("params") or {}
            if method == "tools/call":
                name = params.get("name")
                # Unknown tool -> None (let dispatch 404/INVALID_PARAMS, NOT a 403).
                return TOOL_SCOPE_REQUIREMENTS.get(name) if name else None
            if method == "resources/read":
                return "agent-brain:read"
            if method == "resources/subscribe":
                return "agent-brain:subscribe"
            if method == "prompts/get":
                return "agent-brain:read"
            return None  # initialize, tools/list, etc. -> no scope check

        async def _send_403(self, send, *, required: str) -> None:
            www = (
                f'Bearer error="insufficient_scope", '
                f'scope="{required}", '
                f'resource_metadata="{self.resource_metadata_url}"'
            )
            payload = json.dumps(
                {
                    "error": "insufficient_scope",
                    "error_description": f"Required scope: {required}",
                }
            ).encode()
            await send(
                {
                    "type": "http.response.start",
                    "status": 403,
                    "headers": [
                        (b"content-type", b"application/json"),
                        (b"content-length", str(len(payload)).encode()),
                        (b"www-authenticate", www.encode()),
                    ],
                }
            )
            await send({"type": "http.response.body", "body": payload})
    ```
