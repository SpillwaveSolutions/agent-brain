<objective>
Wire per-tool scope enforcement at the MCP dispatch layer (OAUTH-06 SC#1/#2/#3). Inside
`server.py::call_tool`, `read_resource`, and `handle_subscribe`, read the current request's
token scopes and call `require_scope()` (Plan 01) BEFORE the handler runs — gated to
`AGENT_BRAIN_AUTH=oauth`. On insufficient scope, surface **HTTP 403** with
`WWW-Authenticate: Bearer error="insufficient_scope", scope="<REQUIRED>",
resource_metadata="<PRM url>"` — never 401. Prove it with acceptance tests that mint
minimal-scope tokens and call privileged tools.

Purpose: Middleware "is-authenticated" (Phase 67) cannot scope per-tool — the tool name is
only known at dispatch. This closes design Risk 4 (per-tool scope escalation).
Output: enforcement in server.py (3 dispatch points) + the 403-surfacing layer in http.py +
acceptance tests using read-only tokens against index/admin tools.
</objective>

    The dispatch-layer handler runs INSIDE the StreamableHTTP session (json_response=False,
    stateless=False), so a raised exception would otherwise become a JSON-RPC error inside an
    HTTP 200 envelope. To honor the design's HTTP 403 + WWW-Authenticate contract, add a thin

    Implement `class ScopeEnforcementMiddleware` (or a small ASGI wrapper fn) in http.py:
    ```python
    class ScopeEnforcementMiddleware:
        """Translate dispatch-layer InsufficientScopeError → HTTP 403 insufficient_scope.

        Wraps the MCP app so a per-tool scope failure (server.py::_enforce_scope) becomes
        the design-mandated 403 + WWW-Authenticate header, not a JSON-RPC error in a 200.
        """
        def __init__(self, app, *, resource_metadata_url: str) -> None:
            self.app = app
            self.resource_metadata_url = resource_metadata_url

        async def __call__(self, scope, receive, send) -> None:
            try:
                await self.app(scope, receive, send)
            except InsufficientScopeError as exc:
                await self._send_403(send, required=exc.required)

        async def _send_403(self, send, *, required: str) -> None:
            www = (
                f'Bearer error="insufficient_scope", '
                f'scope="{required}", '
                f'resource_metadata="{self.resource_metadata_url}"'
            )
            body = json.dumps(
                {"error": "insufficient_scope",
                 "error_description": f"Required scope: {required}"}
            ).encode()
            await send({"type": "http.response.start", "status": 403, "headers": [
                (b"content-type", b"application/json"),
                (b"content-length", str(len(body)).encode()),
                (b"www-authenticate", www.encode()),
            ]})
            await send({"type": "http.response.body", "body": body})
    ```
    IMPORTANT — placement so the catch actually fires before the session manager writes the
    response: the StreamableHTTPSessionManager may suppress exceptions and may have already
    started the response. Investigate during implementation (read streamable_http.py
