<objective>
Stand up a real Keycloak ≥22 container in CI and prove the split AS/RS path end-to-end
against it: a Keycloak-issued JWT is accepted via cached JWKS (SC#1), opaque-token
introspection round-trips (SC#2), and a revoked token is rejected on next use (SC#3).

Mirrors the existing `postgres` opt-in skip pattern: a new `keycloak` pytest marker
(registered alongside e2e/e2e_http/contract/stress) gates the slow container-backed tests
so they skip cleanly locally and run in CI. Triggered on PRs touching `agent-brain-mcp/**`
(path filter) AND nightly via e2e-nightly.yml.

CRITICAL (from RESEARCH): Keycloak has NO native RFC 8707 until unreleased 26.8. The
realm bootstrap MUST use the audience scope mapper (`Included Custom Audience =
AGENT_BRAIN_OAUTH_RESOURCE`) to bind `aud` — identical security property. A code comment
in the bootstrap script MUST record this deviation.

CRITICAL (from RESEARCH Pitfall 5/6): GitHub Actions service containers do NOT reliably
support command overrides, and Keycloak's default production `start` fails health checks
(TLS required). The Keycloak container is therefore started with a step-level
`docker run -d ... start-dev` — NOT a `services:` block.

CRITICAL (fixture discovery): pytest only auto-discovers files literally named
`conftest.py`. A file named `conftest_keycloak.py` is NOT loaded for fixture collection,
so the keycloak fixtures must be bridged into the existing `tests/conftest.py` via an
explicit import or every keycloak-marked test fails at runtime with "fixture not found".

Purpose: SC#1–3 are proven against a real IdP, not just mocks; the split-AS path is
gated whenever MCP/OAuth code changes.
Output: keycloak marker, the conftest bridge, the Keycloak-backed test suite + fixtures,
the realm bootstrap script, and the step-level Keycloak CI job (path-filtered PR + nightly).
</objective>
