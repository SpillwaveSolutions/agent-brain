<objective>
Land the split-AS/RS verification code with its fast mock-backed unit tests in a single
wave so that `agent_brain_mcp/oauth/` coverage never dips below 90% as new code lands.

Adds three verification capabilities behind the existing stable `verify_token(token) ->
AccessToken | None` seam:
1. `JwksTokenVerifier` — remote JWKS verification via `PyJWKClient` (5-min TTL +
   kid-miss on-demand refresh) for the split AS/RS topology (OAUTH-11).
2. `IntrospectionTokenVerifier` — RFC 7662 introspection for opaque-token / external-AS
   deployments; `active:false` → None (revocation rides on introspection for free,
   OAUTH-12 SC#2/SC#3).
3. `jti` denylist on the co-located `InMemoryTokenStore` — in-memory revocation list for
   the co-located path (OAUTH-12 SC#3).

Plus the config env vars + a `build_verifier()` selector wired into `http.py` so the
verifier swaps in by config, leaving `LocalRs256Verifier`'s signature and tests untouched.

Purpose: This is the "wire + configure the SDK's split-AS machinery" work. The library
(PyJWKClient) does the JWKS caching; we configure it and adapt it to the SDK protocol.
Output: New verifier classes, jti denylist, config readers, selector wiring, and the
mock-backed test suite that keeps the dev loop green without a Keycloak container.
</objective>
