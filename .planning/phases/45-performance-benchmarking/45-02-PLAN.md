    2. Create `scripts/query_benchmark.py` as a standalone script (not a CLI command). Structure the file with:

       a. **Helper functions** (small, testable, at top of file):
          - `compute_stats(latencies: list[float]) -> dict`: Takes list of latency floats (ms), returns dict with keys `p50`, `p95`, `p99`, `mean`, `min`, `max`, `count`, `qps`. Use `statistics.mean` and manual percentile calculation (sorted list index).
          - `format_mode_status(mode: str, status: str, reason: str = "") -> dict`: Returns `{"mode": mode, "status": status, "reason": reason}`.
          - `build_run_metadata(server_url: str, health_data: dict, chunk_count: int, iterations: int, warmups: int, corpus_folders: list[str]) -> dict`: Returns dict with `date` (ISO 8601), `os` (platform.system() + platform.release()), `python_version` (platform.python_version()), `backend` (from health_data["storage_backend"]), `graph_enabled` (from health_data), `iterations`, `warmups`, `corpus_identity` (corpus_folders), `chunk_count`.

       b. **Preflight checks** function `run_preflight(server_url: str) -> dict`:
          - Check server reachable: GET `{server_url}/health`, fail with clear message if unreachable.
          - Check index ready: GET `{server_url}/health/status`, warn if status != "ready".
          - Check chunk count: GET `{server_url}/query/count`, fail if count == 0.
          - Detect backend type: from health response `storage_backend` field.
          - Detect GraphRAG: from health response (check for `graphrag_enabled` field or similar).
          - Return dict with `server_url`, `backend`, `graph_enabled`, `chunk_count`, `folders` (from folders endpoint if available), `warnings` list.
          - If `docs/` not in indexed folders and not using `--prepare-docs-corpus`, add warning: "Active corpus does not include docs/ -- results are not a baseline".

       c. **Setup mode** function `prepare_docs_corpus(server_url: str, docs_path: str) -> None`:
          - DELETE `{server_url}/index` to reset.
          - POST `{server_url}/index` with `{"folder_path": docs_path}`.
          - Poll `{server_url}/health/status` every 2 seconds until status == "ready" (timeout 300s).
          - Print progress with Rich console.

       d. **Benchmark runner** function `benchmark_mode(server_url: str, mode: str, queries: list[str], iterations: int, warmups: int) -> dict`:
          - Run `warmups` queries (discard results).
          - Run `iterations` queries, collecting: client-side latency (time.perf_counter around httpx POST), server-reported `query_time_ms` from response.
          - If query returns 4xx/5xx, mark mode as unsupported with reason from response.
          - Return dict: `{"mode": mode, "status": "ok"|"unsupported"|"error", "client_stats": compute_stats(...), "server_stats": compute_stats(...), "reason": ""}`.

       e. **Output formatting**:
          - `print_results_table(results: list[dict], metadata: dict) -> None`: Rich table with columns: Mode, Status, p50 (ms), p95 (ms), p99 (ms), Mean (ms), QPS. Include metadata header above table.
          - `build_json_output(results: list[dict], metadata: dict) -> dict`: Full JSON payload with `metadata`, `results`, `unsupported_modes`.

       f. **Main function** with argparse:
          - `--server-url` (default: http://127.0.0.1:8000)
          - `--iterations` (default: 20)
          - `--warmups` (default: 3)
          - `--modes` (default: all 5 modes: vector,bm25,hybrid,graph,multi)
          - `--json` flag for JSON output
          - `--prepare-docs-corpus` flag that calls prepare_docs_corpus before benchmarking
          - `--queries-file` (default: scripts/benchmark_queries.json)

       g. **Main flow**:
          1. Parse args
          2. Load query set from --queries-file
          3. If --prepare-docs-corpus: call prepare_docs_corpus
          4. Run preflight checks, print warnings
          5. For each mode: call benchmark_mode, collect results
          6. Print Rich table or JSON output

       Use `httpx` (sync client) for HTTP calls. Use `rich` for console output. Import both at top level.
