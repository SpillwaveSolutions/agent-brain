The target this workflow drives (Plan 01, already landed + verified in Wave 1):
- `task mcp:framework-matrix` — gated; runs only when FRAMEWORK_MATRIX=1 (or `-- --force`).
- It self-bootstraps each framework: Python via `framework-matrix/bootstrap_venv.sh <fw>` then per-venv pytest; TS via `cd framework-matrix/ts && pnpm install --frozen-lockfile && pnpm test`.
- Therefore the workflow does NOT pre-bootstrap framework venvs — it only needs the TOOLCHAINS on PATH (Task, Python, Poetry, uv, Node, pnpm) plus the local agent-brain packages installed so `agent-brain-serve` / `agent-brain-mcp` resolve (bootstrap_venv.sh installs the local packages into each framework venv, but the seeded server fixture needs them on PATH too — see Phase 61 conftest prerequisite-skip behavior).

Repo CI conventions (from e2e-nightly.yml + pr-qa-gate.yml — match these EXACTLY):
- `uses: actions/checkout@v4`
- `uses: actions/setup-python@v5` with `python-version: '3.11'`
- `uses: snok/install-poetry@v1` with `version: 1.7.1`, `virtualenvs-create: true`, `virtualenvs-in-project: true`
- `uses: arduino/setup-task@v2` with `version: 3.43.3`
- `environment: ci-testing` (the env that holds the OPENAI_API_KEY / ANTHROPIC_API_KEY secrets — used by e2e-nightly.yml and pr-qa-gate.yml)
- e2e-nightly.yml is the closest analogue: it is `schedule: cron '0 6 * * *'` + `workflow_dispatch:`, advisory ("does not block PRs"), runs a slow nightly job, and uses `|| true` to avoid failing the workflow on test failures.

The advisory-only mechanism (CONTEXT decision):
- Trigger ONLY on `schedule` + `workflow_dispatch` — NEVER `push`/`pull_request`. (A workflow that never runs on PR events cannot be a required PR check.)
- Publish a commit status via `actions/github-script@v7` (`github.rest.repos.createCommitStatus`) with `context: 'framework-matrix (advisory)'`, `state: 'success' | 'failure'` derived from the matrix step outcome, and `sha: context.sha`. This makes the result visible on the `main` commit without being a required check.

Node/pnpm setup (the TS leg needs it — NOT present in the existing Python-only workflows; resolve current syntax via context7):
- `actions/setup-node@v4` then `corepack enable pnpm` (framework-matrix/ts/README.md "Prerequisites": pnpm via `corepack enable pnpm`, Node 16.9+; package.json pins `pnpm@10.12.4` in `packageManager`, so corepack will activate that version).
