<objective>
Lock the two highest-risk behaviors with automated tests:

1. **SC#4 / OAUTH-08 confused-deputy (DEDICATED test, the locked invariant):** the
   MCP server's upstream call to `agent-brain-server` (the MCP→REST leg) uses the
   static `X-API-Key: <AGENT_BRAIN_API_KEY>` Bearer and NEVER forwards the client's
   OAuth access token. The MCP→REST header injection lives in
   `agent_brain_mcp/config.py::open_backend_client` →
   `_open_http_client`/`_open_uds_client` (sets `headers["X-API-Key"] = api_key`).
   The test asserts the outgoing REST request carries `X-API-Key` and does NOT
   carry an `Authorization: Bearer <oauth-token>`.

2. **Client-dance behaviors (SC#1/#2/#3):** drive `McpHttpBackend` (or its provider)
   at the SDK level to prove: a 401+WWW-Authenticate runs the dance and retries with
   a Bearer token (SC#1); a second fresh Pattern A backend instance loads the cached
   token from `FileTokenStorage` and does NOT call the redirect_handler again (SC#2);
   an expired-access + valid-refresh token silently refreshes without user
   interaction (SC#3). The SDK `OAuthClientProvider` owns the protocol mechanics —
   these tests mock the AS responses (or inject a fake httpx transport / fake AS)
   and assert the observable outcomes, not re-implement the spec.

Purpose: SC#4 is a locked security invariant — it MUST have its own test. SC#1-3
are the user-facing DoD; this plan is where the goal-backward truths get proven.

Output: two net-new test modules. No production code changes (production was built
in 69-01..69-03).

```python
def _open_http_client(backend_url, timeout, api_key=None) -> httpx.Client:
    client = httpx.Client(base_url=backend_url, timeout=timeout)
    if api_key:
        client.headers["X-API-Key"] = api_key   # the ONLY upstream credential
    return client

def open_backend_client(*, backend=None, backend_url=None, socket_path=None,
                        state_dir=None, timeout=DEFAULT_TIMEOUT) -> tuple[..., httpx.Client]:
    api_key = _resolve_api_key(state_dir)   # AGENT_BRAIN_API_KEY / runtime.json
    ...  # returns (transport, client) with X-API-Key injected
```
Key fact for SC#4: the client's OAuth token lives ONLY on the CLI-side
`McpHttpBackend` (a different component/process). It is sent to the MCP server, and
the MCP server's `open_backend_client` independently sets X-API-Key for the upstream
REST call — the OAuth token has no path into that httpx client. The test proves the
absence of any leak.
