<objective>
Wire the OAuth dance into `McpHttpBackend`. Three moving parts:

1. **Factory** (`oauth_client.py`): `build_oauth_client_provider(url, state_dir)`
   assembles the SDK `OAuthClientProvider` from a `FileTokenStorage` (69-01), a
   `LoopbackCallbackServer` + redirect/callback handlers (69-02), and an
   `OAuthClientMetadata` requesting the full CLI scope union
   (`agent-brain:read agent-brain:index agent-brain:admin`) with the loopback
   `redirect_uri`, `timeout=300.0`, DCR registration (no `client_metadata_url`).

2. **Centralization** (`client.py`): collapse the 17
   `async with streamablehttp_client(self.url) as (read, write, *_):` sites into
   ONE async-context-manager helper `_http_session()` that yields an initialized
   `ClientSession` and injects the optional `auth=<provider | None>`. Build the
   provider lazily once per `McpHttpBackend` instance, gated by an opt-in signal
   (env, mirroring the server's `AGENT_BRAIN_AUTH` — e.g. `AGENT_BRAIN_MCP_AUTH=oauth`).
   When the opt-in is OFF (default), `auth=None` and behavior is byte-identical to
   today. Preserve the lazy `from mcp import ClientSession` import pattern.

3. **Threading** (`transport.py:189`): pass `state_dir` (and let the env-based
   opt-in flow naturally) into the `McpHttpBackend(...)` construction for the
   `transport=mcp, mcp_transport=http` branch.

`McpStdioBackend` MUST NOT be touched (no OAuth on stdio).

Purpose: This is where SC#1 (transparent dance + retry) and SC#2 (token reuse via
persistence) actually happen at the connection layer.

Output: net-new `oauth_client.py`; refactored `client.py`
(`McpHttpBackend._http_session` + lazy provider); threaded `transport.py`; unit
tests proving auth=None default-path is unchanged and the auth path attaches the
provider.
</objective>
