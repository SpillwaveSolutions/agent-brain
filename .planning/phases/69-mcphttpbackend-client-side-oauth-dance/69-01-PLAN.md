<objective>
Create `FileTokenStorage`, the client-side persistence layer that implements the
SDK `mcp.client.auth.oauth2.TokenStorage` Protocol (4 async methods) and writes a
single JSON file at `state_dir/mcp-oauth-tokens.json` with mode `0o600`. It
persists BOTH the `OAuthToken` and the `OAuthClientInformationFull` (the DCR
registration result) so that Pattern A per-call CLI invocations reuse the cached
token AND the cached client registration — no re-registration, no re-dance while
the token is valid.

Purpose: This is the foundation that makes SC#2 (token reuse without re-triggering
the browser) and the silent-refresh path (SC#3, driven by the SDK provider) work.
Without durable, secure persistence the dance would re-run on every Pattern A call.

Output: `agent_brain_mcp/oauth/token_storage.py` (net-new), an export from the
oauth package `__init__.py`, and a thorough unit-test module.
</objective>

    import json
    import logging
    import os
    from pathlib import Path

    from mcp.shared.auth import OAuthClientInformationFull, OAuthToken

    logger = logging.getLogger(__name__)

    TOKEN_FILE_NAME = "mcp-oauth-tokens.json"
    _TOKEN_FILE_MODE = 0o600

    class FileTokenStorage:
        """SDK TokenStorage Protocol impl persisting one JSON file at
        ``state_dir/mcp-oauth-tokens.json`` (chmod 0o600)."""

        def __init__(self, state_dir: Path) -> None:
            self._path = state_dir / TOKEN_FILE_NAME
            self._state_dir = state_dir

        async def get_tokens(self) -> OAuthToken | None: ...
        async def set_tokens(self, tokens: OAuthToken) -> None: ...
        async def get_client_info(self) -> OAuthClientInformationFull | None: ...
        async def set_client_info(self, client_info: OAuthClientInformationFull) -> None: ...
    ```

    File JSON shape (one file holds BOTH keys; either may be absent):

    {
      "tokens": { "access_token": "...", "token_type": "Bearer",

                  "refresh_token": "..." },

    }
    ```

    Implementation requirements:

    - These are `async def` (the Protocol requires it) but the bodies are plain
