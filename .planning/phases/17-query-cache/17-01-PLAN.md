  <action>
Create `agent_brain_server/services/query_cache.py` following the exact pattern from 17-RESEARCH.md Pattern 1. The service must:
- Use `cachetools.TTLCache` with configurable `ttl` and `max_size`
- Use `asyncio.Lock` for writes and invalidation only (reads are lock-free)
- Track `_index_generation` (int, starts at 0), `_hits`, `_misses`
- `make_cache_key(request_params: dict[str, Any]) -> str`: SHA-256(canonical JSON):generation
- `get(key: str) -> Any | None`: lock-free lookup, updates hit/miss counters
- `put(key: str, value: Any) -> None`: async, acquires lock
- `invalidate_all() -> None`: async, acquires lock, increments generation, clears cache
- `get_stats() -> dict[str, Any]`: returns hits/misses/hit_rate/cached_entries/index_generation
- `is_cacheable_mode(mode: str) -> bool`: returns False for "graph" and "multi"
- Module-level singleton: `_query_cache`, `get_query_cache()`, `set_query_cache()`, `reset_query_cache()`

Create `tests/test_query_cache.py` with these test cases:
- `test_cache_miss_returns_none` — fresh cache returns None and increments misses
- `test_cache_hit_returns_cached_result` — after put, get returns the value and increments hits
- `test_cache_key_deterministic` — same params produce same key
- `test_cache_key_different_params` — different params produce different keys
- `test_cache_key_includes_generation` — key changes after invalidate_all()
- `test_invalidate_all_clears_cache` — cached value no longer returned after invalidate
- `test_invalidate_all_increments_generation` — generation counter increases
- `test_graph_mode_not_cached` — is_cacheable_mode("graph") == False
- `test_multi_mode_not_cached` — is_cacheable_mode("multi") == False
- `test_vector_bm25_hybrid_cacheable` — is_cacheable_mode returns True for vector/bm25/hybrid
- `test_get_stats_structure` — get_stats returns all expected keys
- `test_settings_configure_cache` — constructor respects ttl and max_size args
- `test_singleton_pattern` — set/get/reset module-level singleton works
- `test_cache_ttl_expiry` — entry expires after TTL (use TTLCache with ttl=1, sleep 1.1s, or mock timer)
- `test_sorted_list_fields_key_stability` — cache key same regardless of list order in params

Use `pytest.mark.asyncio` for async tests. Do NOT create the asyncio.Lock at module level — only inside `__init__`.
