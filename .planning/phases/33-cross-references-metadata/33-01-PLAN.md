<task type="auto">
  <name>Task 1: Build and run link/path verification script</name>
  <files>scripts/check_doc_links.py</files>
  <action>
Create a Python script `scripts/check_doc_links.py` that:

1. Accepts a list of markdown file globs as arguments (or defaults to the audited doc set)
2. For each markdown file, extracts:
   - All markdown links matching `[text](target)` where target is a relative path (not http/https/mailto)
   - All file paths in code blocks (fenced with ```) that look like real paths (contain `/` and end in common extensions like .py, .md, .ts, .yaml, .toml, .json, .sh, .cfg)
3. For each extracted link target:
   - Resolves it relative to the file containing the link
   - If it contains `#`, splits into file path and anchor
   - Checks the file path exists on disk
   - If anchor present, checks the target file contains a heading matching the anchor (convert heading to slug: lowercase, replace spaces with hyphens, strip punctuation)
4. For each extracted file path reference:
   - Resolves it relative to project root
   - Checks it exists on disk
5. Outputs a JSON report to stdout with structure:
   ```json
   {
     "broken_links": [{"file": "...", "line": N, "link": "...", "target": "...", "reason": "file not found|anchor not found"}],
     "broken_paths": [{"file": "...", "line": N, "path": "...", "reason": "file not found"}],
     "stats": {"files_scanned": N, "links_checked": N, "paths_checked": N, "broken_links": N, "broken_paths": N}
   }
   ```

The script should be runnable with: `python scripts/check_doc_links.py`

Default file set to scan (the audited docs from phases 29-32):
- `docs/*.md` (excluding `docs/plans/` and `docs/design/`)
- `agent-brain-plugin/commands/*.md`
- `agent-brain-plugin/skills/*/references/*.md`
- `agent-brain-plugin/agents/*.md`
- `README.md`, `CLAUDE.md`, `AGENTS.md`, `.claude/CLAUDE.md`

Run the script after creating it and save the JSON output to `/tmp/doc-link-report.json` for Task 2.
  </action>
