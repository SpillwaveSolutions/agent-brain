<objective>
Wire the `AGENT_BRAIN_AUTH` auth-mode toggle (`none` / `basic` / `oauth`) into the MCP
package config layer as a typed `AuthMode`, add the OAuth env settings
(`AGENT_BRAIN_OAUTH_RESOURCE`, `AGENT_BRAIN_OAUTH_ISSUER`), and build the boot-time
startup gate that mirrors the server-side SECURITY-01 contract: validate-at-boot,
exit-on-misconfig (code 2), silent-when-valid. Also add the `get_auth_dependency()`
selector seam that structurally enforces mutual exclusion (exactly one auth path).

This is the OAUTH-09 settings foundation. NO request-path enforcement, NO middleware
wiring, NO route handlers — those are Plan 02 (routes) and Phase 67 (enforcement).

Purpose: Phase 67 wires `RequireAuthMiddleware` keyed off this toggle and reads the
resolved OAuth settings; this plan is the validated foundation it builds on.
Output: Typed `AuthMode` + OAuth settings resolver + `check_auth_startup_gate()` +
`get_auth_dependency()` selector, all in `config.py`, fully unit-tested.
</objective>
