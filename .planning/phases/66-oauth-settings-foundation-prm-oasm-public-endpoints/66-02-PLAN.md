Stand up the public OAuth discovery root: hand-roll the two well-known discovery routes as
plain Starlette `Route` handlers returning config-derived JSON, add them to the
`build_asgi_app()` `routes` list ABOVE the `/mcp` Mount (mount-order is the critical
contract — design doc Risk 3), wire the Plan 01 startup gate into the top of
`build_asgi_app()`, and prove with tests that both endpoints return HTTP 200 with NO
Authorization header — written so the test still passes once Phase 67 adds
`RequireAuthMiddleware` (proving the routes live outside the future middleware scope).

Also resolve the `/mcp/subscriptions` audit flag: confirm whether that debug endpoint
actually shipped (a scout found only /healthz + /mcp in http.py); record the finding so
the design-doc audit item is not silently dropped.

This is OAUTH-02 (PRM + path-suffixed variant) + OAUTH-03 (OASM with S256). NO token
issuance, NO /authorize|/token|/register|jwks routes, NO request-path enforcement —
those are Phase 67. The OASM advertises forward-reference URIs for Phase-67 routes; that
is correct and MUST be documented as such in code comments.

Purpose: compliant MCP SDK clients abort silently on a malformed discovery document, so
the correctness bar (exact field names, exact S256 advertisement, mount order) is high.
Output: oauth_metadata.py builders + two well-known routes wired in http.py + the
mount-outside-middleware acceptance test + the subscriptions audit note.
