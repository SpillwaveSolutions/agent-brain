<objective>
Eliminate the #178 class of failure where a native kuzu SIGSEGV during sustained
GraphRAG indexing kills the entire server process. Because a SIGSEGV is a native
process death that Python try/except cannot catch, the kuzu write/commit path is
isolated OUT-OF-PROCESS: a native child crash becomes a catchable non-zero child
exit, not a parent death. On graph failure the indexing job degrades to "no graph
this run" — vector + BM25 indexing for that job still commit — and surfaces a clear
structured error. The configured `graphrag.store_type` is never silently rewritten;
`simple` stays the documented manual fallback.

Purpose: server stays up under any kuzu-native failure (Phase 64 Success Criterion 1).
Output: out-of-process graph build isolation + graceful per-job degradation + tests.
</objective>
