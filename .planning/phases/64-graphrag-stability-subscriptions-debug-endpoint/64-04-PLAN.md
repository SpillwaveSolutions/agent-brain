<objective>
Close #194 (deferred from v10.2 VAL-02): ship a `GET /mcp/subscriptions` debug
endpoint on the MCP Streamable-HTTP Starlette app, mounted alongside `/healthz` in
`http.py`. It returns 200 + JSON describing the current active subscription state —
transport, server uptime, active count, and a list of subscriptions (each with a
TRUNCATED session id, uri, cadence, started_at, last_notified_at) — with NO token
(loopback-only, same trust model as `/healthz`). The route reads a read-only
`snapshot()` method on `SubscriptionManager` rather than reaching into `_tasks`
directly. Under stdio transport there is no HTTP listener, so the endpoint simply
does not exist (no shim).

Purpose: operators can curl live subscription state for diagnosis (Phase 64 Success Criterion 4).
Output: SubscriptionManager.snapshot() + /mcp/subscriptions route + tests.
</objective>
