<objective>
Close #184 bug 1: when kuzu opens cleanly but the live graph is STALE after an
`AGENT_BRAIN_JOB_TIMEOUT` rollback, give operators an explicit way to replay the
latest snapshot back into kuzu. Ship `agent-brain graph restore-from-snapshot
[--snapshot PATH] [--dry-run] [--yes]` (confirm-by-default; `--yes` for CI;
`--dry-run` reports the plan without mutating). Make `agent-brain doctor` WARN on the
stale-graph condition instead of reporting OK, and let `agent-brain doctor --fix`
perform the restore via the existing safe-idempotent `--fix` framework. Reuse the
existing `GraphSnapshotManager` + the `_restore_from_snapshot_if_available` restore
machinery — expose a public `restore_from_snapshot(snapshot_path)` primitive for it.

Purpose: operators can diagnose and restore a stale graph (Phase 64 Success Criterion 2).
Output: graph restore CLI + public restore primitive + doctor stale-graph check/--fix + tests.
</objective>
