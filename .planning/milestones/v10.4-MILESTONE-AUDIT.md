## Cross-Phase Integration

**Score: 14/14 exports wired, 0 orphaned, 0 broken runtime flows.**

The central OAuth chain is verified end-to-end:

1. **Discovery reachable unauthenticated** (OAUTH-02/03): PRM, OASM, JWKS, /healthz,
   /mcp/subscriptions Routes sit in `exempt_routes` BEFORE the /mcp Mount in `build_asgi_app()`;
   Starlette first-match guarantees they serve in all modes even with RequireAuthMiddleware wired.
2. **Verifier selection behind one seam** (OAUTH-09/04/05/11/12): `build_verifier()` selects
   `JwksTokenVerifier` (JWKS_URI set) → `IntrospectionTokenVerifier` (INTROSPECTION_URL set) →
   `LocalRs256Verifier` (default), all implementing `verify_token() -> AccessToken | None`. Wired
   at http.py via `build_verifier(issuer_override=issuer)`; no remaining `build_local_verifier` call.
3. **Middleware composition** (OAUTH-05/06): `AuthenticationMiddleware → RequireAuthMiddleware →
   ScopeEnforcementMiddleware → mcp app` (Phase 68-corrected order), proven by 61 acceptance tests.
4. **Confused-deputy invariant** (OAUTH-08): the client OAuth token (via `OAuthClientProvider` on the
   MCP URL leg) never reaches the MCP→REST client (`_open_http_client` sets only `X-API-Key`);
   `test_oauth_confused_deputy.py` asserts the token string is absent from every upstream header.
5. **Keycloak CI flow** (OAUTH-11/12): `conftest_keycloak.py` fixtures bridged into `conftest.py`;
   step-level `docker run ... start-dev` job; binding `--cov-fail-under=90` gate. Structurally sound
   (executes in CI with a live container; opt-in/deselected locally).

## Tech Debt (non-blocking)

**Phase 67 — http.py docstring drift (cosmetic):** module-level docstring (lines 19-22) shows the
Phase 67-era middleware nesting; the actual code and the ScopeEnforcementMiddleware class docstring
show the correct Phase 68-fixed order. Documentation-only; zero runtime impact. ~2-line fix.

**Phase 70 — no public /revoke endpoint (deliberate deferral):** revocation is provided via
introspection (`active:false`) and the co-located jti denylist; a public RFC 7009 /revoke route is
deferred to v10.4.1 and documented in `agent-brain-mcp/docs/SPLIT_AS_RS.md`. Not a DoD gap (OAUTH-12
DoD is "introspection + revocation supported", which is met).

**Phase 66 — basic-mode /mcp mount unwrapped (pre-existing observation):** under
`AGENT_BRAIN_AUTH=basic` the /mcp mount carries no RequireAuthMiddleware; enforcement relies on the
existing `AGENT_BRAIN_API_KEY` Bearer mechanism. Not a regression; flagged for operator awareness.
