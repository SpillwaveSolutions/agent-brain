<objective>
Create a config validation engine and `agent-brain config validate` CLI command that checks config.yaml against the known schema, reporting errors with field paths, line numbers, and fix suggestions.

Purpose: Users currently have no way to verify their config.yaml is correct before starting a server. Invalid configs cause cryptic Pydantic errors at server startup. This plan gives them a fast, offline validation tool.

Output: `config_schema.py` validation module, `validate` subcommand on the config group, comprehensive tests.
</objective>

Add a `validate` subcommand to `config_group` in `agent-brain-cli/agent_brain_cli/commands/config.py`:

```python
@config_group.command("validate")
@click.option("--file", "config_file", type=click.Path(exists=True), default=None,
              help="Path to config file (default: auto-detect)")
@click.option("--json", "json_output", is_flag=True, help="Output as JSON")
def validate_config(config_file: str | None, json_output: bool) -> None:
    """Validate config.yaml against the Agent Brain schema.

    Checks for unknown keys, invalid provider values, deprecated fields,
    and type errors. Reports line numbers and fix suggestions.

    \b
    Examples:
      agent-brain config validate
      agent-brain config validate --file ./my-config.yaml
      agent-brain config validate --json
    """
```

Implementation:
1. If `--file` provided, use `Path(config_file)`. Otherwise call `_find_config_file()`.
2. If no config found, print "[yellow]No config file found. Nothing to validate.[/]" and `sys.exit(0)`.
3. Call `validate_config_file(path)` from config_schema module.
4. If no errors:
   - Rich mode: print "[green]Config is valid[/] ({path})" and `sys.exit(0)`
   - JSON mode: print `{"valid": true, "config_file": str(path), "errors": []}` and `sys.exit(0)`
5. If errors:
   - Rich mode: print `format_validation_errors(errors)` via `console.print` and `sys.exit(1)`
   - JSON mode: print `{"valid": false, "config_file": str(path), "errors": [{field, message, line_number, suggestion}...]}` and `sys.exit(1)`
