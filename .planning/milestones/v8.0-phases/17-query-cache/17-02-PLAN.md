```markdown
### Query Cache

| Variable | YAML Path | Default | Description |
|----------|-----------|---------|-------------|
| `QUERY_CACHE_TTL` | `query_cache.ttl` | `300` | Time-to-live for cached query results in seconds. Set to 0 to disable TTL (size-based eviction only). |
| `QUERY_CACHE_MAX_SIZE` | `query_cache.max_size` | `256` | Maximum number of query results to cache. When full, least-recently-used entries are evicted. |

Notes:
- Query cache is in-memory only (no disk persistence). Cache is empty after server restart.
- `graph` and `multi` query modes are never cached (non-deterministic LLM extraction).
- Cache is automatically invalidated on every successful reindex job completion.
```
