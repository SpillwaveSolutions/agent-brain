  <action>
Create `docs/INJECTOR_PROTOCOL.md` documenting the content injection system (INJECT-08):

**Structure:**

1. **Overview** â€” What content injection does: enrich chunk metadata during indexing with custom data via Python scripts or JSON files.

2. **Quick Start** â€” Minimal example:
   ```bash
   # Using a Python script
   agent-brain inject --script enrich.py /path/to/docs

   # Using folder metadata JSON
   agent-brain inject --folder-metadata metadata.json /path/to/docs

   # Both combined
   agent-brain inject --script enrich.py --folder-metadata metadata.json /path/to/docs

   # Validate before indexing
   agent-brain inject --dry-run --script enrich.py /path/to/docs
   ```

3. **The `process_chunk` Protocol** â€” Detailed spec:
   - Function signature: `def process_chunk(chunk: dict) -> dict`
   - Input: flat dict from `ChunkMetadata.to_dict()` with keys: `chunk_id`, `source`, `file_name`, `chunk_index`, `total_chunks`, `source_type`, `created_at`, `language`, `heading_path`, `section_title`, `content_type`, `symbol_name`, `symbol_kind`, `start_line`, `end_line`, `section_summary`, `prev_section_summary`, `docstring`, `parameters`, `return_type`, `decorators`, `imports`, plus any `extra` keys.
   - Output: dict with enriched metadata. New keys are stored in `ChunkMetadata.extra`. Existing fixed keys (chunk_id, source, etc.) cannot be overridden â€” only new keys are captured.
   - Values MUST be scalars: `str`, `int`, `float`, `bool`. Lists and dicts are rejected with a warning (ChromaDB constraint).
   - Exceptions: per-chunk exceptions are caught, logged, and the original chunk is preserved. The indexing job continues.

4. **Example Scripts** â€” Two complete working examples:
   - `enrich.py`: Add project tags, conditional enrichment based on source_type.
   - `classify.py`: Classify chunks by content (e.g., add `doc_category` based on file extension).

5. **Folder Metadata JSON** â€” Spec:
   - Must be a JSON object (dict at root level).
   - All values must be scalars.
   - Merged into every chunk before script injection.
   - Example JSON shown.

6. **Dry-Run Mode** â€” How `--dry-run` works:
   - Loads 3 sample files from the folder, chunks them, applies injection.
   - Returns report with chunk count and sample of injected keys.
   - Does NOT enqueue a job or modify the index.

7. **Best Practices:**
   - Keep `process_chunk` pure (no side effects, no network calls).
   - Avoid top-level code in scripts (runs on import).
   - Use `--dry-run` before production indexing.
   - Use folder metadata for static project-wide tags.
   - Use scripts for conditional or computed enrichment.

8. **Limitations:**
   - Scripts run in the server process (same machine, same Python).
   - No sandboxing â€” scripts have full Python access.
   - Values must be flat scalars (no nested dicts/lists).
   - Script path must be accessible from the server process.

Keep the doc concise (~150 lines). No emojis. Use code blocks for all examples.
